## 🎯 MVP版本(0.9.0)

### 目标与原则
**核心验证点**:
- 用户能否完成"记录-反馈-改善"闭环?
- 核心数据流程是否通畅?
- 用户体验是否友好?

**MVP范围**:
只做最核心的3个功能,其他都砍掉!

### MVP功能列表

#### 1. 健康档案(最简版)
- [ ] 基础信息录入 (身高、体重、目标)
- [ ] 体重记录 (仅记录数值,无图表)
- [ ] BMI自动计算
- [ ] 数据保存本地

#### 2. 餐食打卡(最简版)
- [ ] 三餐打卡按钮 (早餐、午餐、晚餐)
- [ ] 拍照/相册选择图片
- [ ] 简单文字备注 (20字以内)
- [ ] 打卡日历查看

#### 3. 核心数据看板
- [ ] 今日完成情况展示
- [ ] 体重趋势 (简单曲线图)
- [ ] 打卡进度展示

### MVP技术架构
- **前端**: Uni-app (支持多端)
- **后端**: Spring Boot + MyBatis Plus
- **数据库**: MySQL
- **存储**: 七牛云/腾讯云OSS
- **部署**: 腾讯云轻量应用服务器

### MVP开发冲刺规划(2周)

**冲刺0: 项目准备 (1天)**
- 技术选型确认
- 项目脚手架搭建
- 开发环境配置
- 代码规范制定

**冲刺1: 健康档案模块 (3天)**
- 用户基础信息录入界面
- 体重记录功能
- BMI计算逻辑

**冲刺2: 餐食打卡模块 (3天)**
- 三餐打卡界面
- 图片选择/上传功能
- 打卡记录存储
- 简单日历展示

**冲刺3: 数据看板+集成 (2天)**
- 主页数据展示

**冲刺4: 测试与发布 (2天)**

## 🏗️ 技术架构与开发规范

### 技术栈选择

#### 前端技术栈
- **核心框架**: Uni-app (Vue3 + TypeScript)
- **UI框架**: uView UI / TDesign Uni-app版
- **图表库**: uCharts / ECharts
- **状态管理**: Pinia
- **路由**: Vue Router
- **网络请求**: axios + 拦截器封装
- **本地存储**: uni.setStorage 统一封装
- **打包工具**: HBuilderX / Webpack

**优势**:
- 一套代码多端发布 (iOS、Android、小程序、H5)
- 成熟的生态和社区
- 开发效率高,学习成本低

#### 后端技术栈
- **核心框架**: Spring Boot 3.x
- **数据库**: MySQL 8.0 + Redis
- **ORM框架**: MyBatis Plus
- **权限认证**: Spring Security + JWT
- **缓存**: Spring Cache + Redis
- **消息队列**: RabbitMQ (异步处理)
- **文件存储**: 腾讯云COS / 七牛云OSS
- **API文档**: SpringDoc OpenAPI
- **监控**: Spring Boot Actuator + Prometheus + Grafana

#### 部署架构
```
用户 → CDN (静态资源) → Nginx → 后端集群 → MySQL主从
                    ↓
                    Redis缓存层
                    ↓
                    消息队列
                    ↓
                    文件存储
```

### 开发规范

#### 代码规范
1. **命名规范**
   - 类名: PascalCase (UserService)
   - 方法名: camelCase (getUserInfo)
   - 变量名: camelCase (userName)
   - 常量: UPPER_SNAKE_CASE (MAX_RETRY_COUNT)
   - 文件命名: kebab-case (user-profile.vue)

2. **目录结构**
   ```
   src/
   ├── api/           # API接口定义
   ├── components/    # 公共组件
   ├── pages/        # 页面组件
   ├── store/        # 状态管理
   ├── utils/        # 工具函数
   ├── static/       # 静态资源
   └── types/        # TypeScript类型定义
   ```

3. **Git规范**
   - 分支策略: Git Flow
   - 提交信息格式: `<type>(<scope>): <subject>`
   - 类型: feat/fix/docs/style/refactor/test/chore
   - 示例: `feat(user): 添加用户注册功能`

#### 数据库设计规范

1. **表命名规范**
   - 业务表: `t_<业务名>` (t_user, t_diet_record)
   - 关联表: `tr_<主表>_<关联表>` (tr_user_follow)
   - 数据字典: `td_<字典名>` (td_diet_type)

2. **字段规范**
   - 主键: `id` (bigint unsigned auto_increment)
   - 创建时间: `created_at` (datetime)
   - 更新时间: `updated_at` (datetime)
   - 软删除: `is_deleted` (tinyint default 0)
   - 状态字段: `status` (tinyint) + 状态枚举类

3. **索引策略**
   - 主键索引: 必须
   - 外键索引: 关联查询字段
   - 组合索引: 常用查询条件
   - 唯一索引: 业务唯一性约束

#### API设计规范

1. **RESTful规范**
   ```
   GET    /users          # 获取用户列表
   POST   /users          # 创建用户
   GET    /users/{id}     # 获取单个用户
   PUT    /users/{id}     # 更新用户
   DELETE /users/{id}     # 删除用户
   ```

2. **响应格式**
   ```json
   {
     "code": 200,
     "message": "success",
     "data": {},
     "timestamp": 1711872000
   }
   ```

3. **错误码规范**
   - 2xx: 成功
   - 4xx: 客户端错误
   - 5xx: 服务端错误
   - 业务错误码: 10000-19999
